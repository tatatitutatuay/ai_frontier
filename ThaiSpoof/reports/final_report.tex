\begin{abstract}
This project studies Thai voice anti-spoofing as a binary classification task: each utterance is classified as either genuine speech or spoofed speech. The experiment uses balanced subsets of ThaiSpoof-style genuine and spoofed audio, extracts cepstral acoustic features, and compares lightweight CNN-based models that can run on a MacBook Air M4 with 16 GB RAM. On the selected Mac-sized subset, LFCC with a small CNN, LFCC with a lightweight ResNet, and MFCC with a small CNN all achieved 1.000 test accuracy, 1.000 F1-score, and 0.000 EER. These results show that the selected subset is highly separable, but they should be interpreted carefully because perfect performance may reflect dataset-source artifacts as well as true spoofing cues.
\end{abstract}

\section{Introduction}

Voice biometric systems are vulnerable to spoofing attacks from synthetic speech, voice conversion, and other generated audio. This project studies voice anti-spoofing as a binary classification problem: classify each utterance as either genuine speech or spoofed speech.

The goal is to build a practical experiment pipeline that can run on a MacBook Air M4 with 16 GB RAM while still producing results suitable for comparison in a biometrics mini-project.

\section{Dataset}

The project uses ThaiSpoof-style Thai speech data from the downloaded local folders under \texttt{data/raw/}:

\begin{itemize}
  \item \texttt{data/raw/genuine/} for bona fide human speech
  \item \texttt{data/raw/Corpus-Spoof-VAJA/} for spoofed speech
\end{itemize}

The full available local dataset contains 4,583 genuine files and 4,583 spoof files. A balanced subset was used to keep training practical on local CPU hardware.

\section{Feature Extraction}

Two cepstral feature types were compared:

\begin{itemize}
  \item \textbf{LFCC}: linear-frequency cepstral coefficients. LFCC is commonly used in anti-spoofing research because linear-frequency spacing can preserve high-frequency artifacts that help reveal synthetic speech.
  \item \textbf{MFCC}: mel-frequency cepstral coefficients. MFCC is a common speech-processing baseline and was used as an optional comparison.
\end{itemize}

Each audio file was converted into a fixed-size feature matrix. Short utterances were repeated to reach the target frame length, while long utterances were truncated.

\section{Discussion}

All three experiments achieved perfect test-set performance on the selected balanced subset. This means the models separated the downloaded genuine and spoof files without any observed classification errors.

The result is strong, but it should be interpreted carefully. Perfect performance can happen when spoofed speech has clear artifacts, but it can also indicate that the classifier is learning dataset-source differences rather than general spoofing cues. For example, genuine files and spoof files may differ in generation pipeline, file structure, acoustic channel, duration distribution, or other metadata-related artifacts.

The LFCC baseline and ResNet-lite improvement produced the same final test metrics. Because the lightweight CNN already reached perfect accuracy and EER, the deeper residual model did not show measurable improvement on this subset. MFCC also reached the same score, suggesting that this local subset may be easy to separate with standard cepstral features.

\section{Limitations}

\begin{itemize}
  \item The full dataset was not used; experiments used a balanced subset suitable for a MacBook Air M4.
  \item The test split comes from the same downloaded data source as the training split, so it may not measure cross-dataset generalization.
  \item Perfect scores may reflect source or synthesis artifacts specific to this dataset subset.
  \item Fixed-size feature matrices truncate long utterances and repeat short utterances.
  \item Training used local CPU-friendly model sizes rather than large anti-spoofing architectures.
\end{itemize}

\section{Conclusion}

The project successfully built a practical Thai voice anti-spoofing pipeline using balanced genuine and spoof subsets, cached acoustic feature extraction, and lightweight CNN-based classifiers.

On the selected Mac-sized subset, all tested configurations reached 100\% test accuracy, 1.000 F1-score, and 0.000 EER. The best practical choice is therefore the LFCC + \texttt{small\_cnn} baseline because it achieved the same performance as the deeper model with lower complexity.

Future work should test on a larger and more diverse split, evaluate cross-source generalization, and include additional spoof types such as subtle F0 modification or small pitch-shift attacks to check whether the perfect subset performance remains robust.
